# Auditor 严格输出模板

## 目的

为确保 T3 审计结果的**可复现性**和**稳定性**，每个 Auditor 必须按本模板输出评分表格。表格格式固定，不得省略或篡改列结构。

---

## 通用规范

- **必须**：每个 Auditor 输出包含「评分表格」的完整报告
- **表格列**：检查项ID | 检查项名称 | 满分 | 得分 | 简短理由（≤50字）
- **简短理由**：一句话概括，引用证据来源（如 S1、S2）
- **禁止**：空单元格、省略检查项、合并/拆分固定列

## 输出格式要求

### 1. Markdown 报告必须包含

每个 Auditor 的报告（无论是 `.md` 还是 `.json`）**必须**包含上述评分表格的完整填写版本。

### 2. JSON 报告对应结构

JSON 报告中的 `checklist_items` 必须与表格一一对应，且 `reason` 字段不得超过约 50 字（中文）或 80 字符（英文），可另用 `evidence` 数组存放详细引用。

```json
{
  "checklist_items": {
    "1.1": {
      "score": 6,
      "max_score": 10,
      "reason": "痛点清晰但无独立验证；S2",
      "evidence": ["S2: 宣称用途"]
    }
  }
}
```

### 3. extract_for_report（必含，供 99-audit-report 聚合）

每个 Auditor 的 JSON 报告**必须**包含 `extract_for_report` 对象，供报告生成步骤聚合文字分析内容，避免每次重写导致不一致。

```json
{
  "extract_for_report": {
    "litmus_test_answer": "否",
    "litmus_test_reason": "一两句解释，说明 Litmus Test 结论依据",
    "strengths_bullets": ["优势1；引用 S2", "优势2；引用 S3"],
    "weaknesses_bullets": ["劣势1；引用 S2", "劣势2"],
    "key_evidence": ["检查项 1.2: 具体证据; S2", "检查项 3.2: 具体证据; S2"],
    "critical_issues": []
  }
}
```

- **Tool/Toy**：`critical_issues` 可省略或为空数组
- **Trash**：`critical_issues` 必填，列出关键问题（火眼金睛触发项等）
- 内容需具体、可引用，便于直接用于 99-audit-report 的文字分析部分

### 4. 可复现性验证

- 表格行数固定：Tool 12 行+合计、Toy 12 行+合计、Trash 14 行+合计
- 列名固定：检查项ID、检查项名称、满分、得分、简短理由
- 任一单元格空缺视为报告不完整
- `extract_for_report` 不得省略

## 与 SKILL 流程的集成

执行三 Auditor 专项评分时：

1. 读取对应的 auditor 指南（tool/toy/trash-auditor.md）
2. **必读**本模板文件 `auditor-templates.md`
3. 按模板表格逐项评分并填写
4. 输出时先输出**完整填写的表格**，再输出 JSON（如有）
